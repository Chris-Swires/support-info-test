Translation Dashboard Integration

Building a seamless bridge between the SNOMED CT authoring experience and a dedicated translation dashboard. This overview highlights how editors will move between applications, how translated content flows back into authoring tasks, and how AI-assisted translation becomes a first-class capability.

Cross-App Navigation AI Enabled Author Experience

Integration Highlights

What authors gain the moment we connect the Authoring UI to the Translation Dashboard.

  • ◆Single-click navigation from the Authoring UI into the Translation Dashboard.
  • ◆Project-aware dropdown inside the translation experience for importing back into authoring tasks.
  • ◆Integration powered via simplex-api today, with a path to a dedicated Import Service.
  • ◆Clear hand-off: translate in the dashboard, quality assure in authoring.

How the Applications Connect

The Authoring UI links externally to a new Translation Dashboard. Within that dashboard, authors select the relevant project and push approved translations back to the originating task. The integration diagram shows the current simplex-api bridge and an optional import service for future scale.

Integration Flow

Authors initiate translation from the Authoring UI. The external dashboard provides a focused translation workspace, yet stays aware of SNOMED CT projects and tasks. When translation is complete, the set is imported back into the task using an API bridge.

  • •Outbound link: Author jumps directly from a task to the Translation Dashboard.
  • •Project context: Dashboard dropdown mirrors Authoring UI projects to keep translations targeted.
  • •Import path: Today via simplex-api, with a future-friendly import service for bulk or AI generated translations.
  • •Downstream: Translations land in Snowstorm through existing publishing channels.

Managing Translation Sets

Selecting a set reveals the worklist, team status, and entry points into the detailed translation tool. Authors can create new sets, see changes since creation, and open the translation tool in context.

  • ◆Set metadata keeps editors aligned on language code, translation counts, and review cadence.
  • ◆"Open in Translation Tool" deep-links directly to the corresponding Weblate queue.
  • ◆Embedded members list surfaces the next concepts to translate, priming authors before they dive in.

Translation Sets Dashboard

Authors arrive in the dashboard with a project-centric view of every translation set. Progress bars expose velocity, while the navigation automatically scopes work to the active code system.

  • ◆At-a-glance metrics surface translated, untranslated, and overall progress for each set.
  • ◆Set chips act as deep links, taking the author directly into focused translation workflows.
  • ◆Filters mirror SNOMED CT projects and reference sets so authors always know the scope of work.

Translating with Weblate

Editors translate individual strings with rich concept context, AI assistance, and streamlined review. The translation dashboard deep-links to this view so authors remain in the same clinical context throughout.

  • ◆Concept descriptions, parents, and attributes display alongside the editable preferred term.
  • ◆Automatic suggestions blend translation memory, AI prompts, and other languages.
  • ◆Review states and action buttons (Save, Suggest, Skip) keep teams moving quickly.

Beyond the Dashboard: Standardised AI Translation

The work happening inside this repository lays the groundwork for deeper alignment. Our future state brings the Translation Dashboard, Authoring UI, and AI services into a consistent pipeline.

1. Unified AI Patterns

Reuse the authoring-ui AI prompt orchestration for translations, ensuring consistent logging, safety checks, and user controls.

2. Service Abstraction

Extend the upcoming import service to handle AI generated translations, aligning terminology validation and release workflows with existing simplex-api endpoints.

3. Continuous Feedback Loop

Feed authoring-ui review outcomes back to the Translation Dashboard and AI models, forming a standardised feedback loop for translation quality.

Next Milestones

The integration outlined here is the launch point. The next iteration embeds the Translation Dashboard context deeper into the Authoring UI, allowing standard prompts, reusable golden sets, and automated QA to be shared between products.
